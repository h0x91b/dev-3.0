# dev3 iOS App ‚Äî Implementation Plan

Executes [DESIGN.md](DESIGN.md). Written for a fleet of parallel agents: every ticket is self-contained, names its dependencies, and has acceptance criteria an agent can verify itself. **Read DESIGN.md first; it is the source of the decisions referenced here.**

## Ground rules for the fleet

- **Contract first**: no client code lands against an undocumented protocol behavior. `docs/ios/PROTOCOL.md` (produced by T0.2) is the single reference; if it and the server disagree, fix the doc + contract test in the same PR.
- **Repo layout**: everything under `ios/`. Xcode project generated by **XcodeGen** from `ios/project.yml` (never commit `.pbxproj` edits by hand). Logic lives in SPM packages so `swift test` (macOS, no simulator) verifies it:

```
ios/
  project.yml               # XcodeGen spec ‚Äî the only project definition
  App/                      # thin app target: entry, scenes, DI wiring
  Packages/
    Dev3Kit/                # SessionClient, RPCClient, PTYClient, models, store
    Dev3TerminalKit/        # SwiftTerm wrapper, input encoding, composer logic
    Dev3UI/                 # Theme (tokens), shared components
  scripts/                  # gen-theme.ts (token extraction), CI helpers
  README.md                 # build/run/test instructions for agents & humans
```

- **Verification**: each ticket's PR must pass `swift test` for touched packages, `xcodebuild build` for the app scheme when the app target is touched, and repo-standard `bun run lint` + `bun run test` when backend files are touched. UI tickets: attach a simulator screenshot to the PR.
- **Testing style mirrors the repo's**: unit-test pure logic exhaustively (FSMs, framing, encoding ‚Äî the web equivalents `remote-session.ts` and `rpc.ts` are already unit-tested; port their test cases), thin UI smoke tests only.
- **Backend tickets** follow all existing repo rules (changelog entry, decision records, `~/.dev3.0/` layout invariants, graceful degradation).

## Phase 0 ‚Äî Foundations

*T0.1 blocks everything under `ios/`. T0.2 and T0.4 are backend-side and run in parallel with it.*

**T0.1 ‚Äî Scaffold `ios/`** (M)
XcodeGen spec, the three SPM packages with placeholder tests, thin app target that builds and shows a placeholder screen, SwiftLint + swift-format configs, `ios/README.md`, GitHub Actions workflow (macOS runner: `swift test` per package + `xcodebuild -scheme Dev3 build` for iOS Simulator). Bundle SwiftTerm via SPM. Bundle JetBrainsMono Nerd Font Mono (copy from `src/mainview/assets/fonts/`).
*Accept:* CI green on the workflow; `xcodegen && xcodebuild build` works from clean clone; `swift test` runs in each package.

**T0.2 ‚Äî Protocol contract: `docs/ios/PROTOCOL.md` + contract tests** (M, backend repo)
Write the wire spec from code (not memory): `/auth/exchange`, `/auth/refresh` (status codes, cookie attrs), `/rpc` framing (request/response/message shapes, timeout, unknown-method error), `/pty` (session id forms, text framing, resize OSC `\x1b]resize;<cols>;<rows>\x07`, close codes 4000‚Äì4003, min-size negotiation behavior, OSC52 out-of-band), `/health`, `/instance` (after T0.4), push-event catalog with payload types (from `types.ts:3198-3318`), reconnection semantics (no replay ‚Üí refetch). Add vitest e2e tests against a headless server instance that pin each fact (extend existing e2e patterns; keep out of the fast suite if slow).
*Accept:* tests green in `bun run test:full`; PROTOCOL.md reviewed against `remote-access-server.ts` + `pty-server.ts` line references.

**T0.3 ‚Äî `Dev3UI` theme port** (S, depends T0.1)
`scripts/gen-theme.ts`: parse `src/mainview/index.css` tokens (both themes) + `STATUS_COLORS`/`STATUS_COLORS_LIGHT` from `src/shared/types.ts` ‚Üí generate `Theme.swift`. Commit both script and output; CI check that regenerating produces no diff. Nerd Font glyph constants for the icons the web uses.
*Accept:* generated Swift compiles; a `ThemePreview` SwiftUI preview shows both palettes; regen check wired into iOS CI job.

**T0.4 ‚Äî Backend: `GET /instance` + Bonjour** (S‚ÄìM, backend repo)
Authenticated-optional `GET /instance` returning `{instanceId, name, appVersion, protocolVersion}` (instanceId: new persisted UUID, additive file under `~/.dev3.0/` ‚Äî never rename/move anything existing). Advertise `_dev3._tcp` with the instanceId TXT record while the remote server runs; feature-gated, silent no-op if the mDNS API is unavailable. Decision record + changelog entry.
*Accept:* unit tests for the handler; `dns-sd -B _dev3._tcp` shows the instance when remote is up; older clients unaffected.

## Phase 1 ‚Äî Core kit (all parallel after T0.1/T0.2; the fleet's widest fan-out point)

**T1.1 ‚Äî `Dev3Kit.SessionClient`** (M)
Port `src/mainview/remote-session.ts` (states idle‚Üíauthenticating‚Üíconnecting‚Üíconnected‚Üíreconnecting‚Üíexpired) with its unit tests. Cookie handling: store the `dev3_session` value from `Set-Cookie`, resend as a `Cookie:` header manually (do not rely on shared `HTTPCookieStorage` across origins). Keychain-backed multi-server store `{origin, sessionToken, name, instanceId}`. 15-min rolling refresh; on-close probe of `/auth/refresh` (401/403 ‚Üí expired, else backoff 2s‚Üí15s); `kick()` API for scene-foreground / `NWPathMonitor` path change.
*Accept:* FSM unit tests cover every transition incl. the consumed-QR-falls-back-to-refresh path; integration test against a real headless server (macOS `swift test`, spawning `dev3 remote --no-tunnel --static-code test1234` ‚Äî document the recipe in ios/README.md).

**T1.2 ‚Äî `Dev3Kit.RPCClient` + models + push dispatch** (L)
WebSocket JSON framing per PROTOCOL.md: monotonic ids, pending map, 120s timeout, queue-unsent-until-open, reject-in-flight-on-close. Typed async methods for the **v1 subset** (board: `getProjects`, `getAllProjectTasks`, `getTasks`; task: `createTask`, `editTask`, `renameTask`, `moveTask`, `setTaskPriority`, `toggleTaskWatch`, `setTaskLabels`, label CRUD, note CRUD, `setUserOverview`, `deleteTask`, `spawnVariants`; terminal: `getPtyUrl`, `getProjectPtyUrl`, `resumeTask`, `restartTask`, `tmuxAction`, `tmuxPaneNavigate`, `tmuxWindowNavigate`, `tmuxPaneCount`, `tmuxKillPane`, `getTerminalPreview`; git/PR: `getBranchStatus`, `getTaskDiff`, `refreshTaskPrStatus`, `pushTask`, `createPullRequest`, `pullProjectMain`; agents: `getAgents`; settings: `getGlobalSettings`, `getAppVersion`; context: `setActiveContext`, `setWindowForeground`, `setTerminalFocus`, `ping`, `logRendererError`). Codable models annotated with `types.ts` line refs. Push dispatcher (typed enum) for the v1 events listed in DESIGN.md ¬ß8.
*Accept:* framing unit tests (correlation, timeout, queueing, malformed frames); round-trip integration test against headless server for 5+ representative methods; model decode tests from captured real payloads.

**T1.3 ‚Äî `Dev3Kit.PTYClient`** (M)
Connect `/pty?session=<id>` with cookie; deliver output chunks; `send(bytes)`; `resize(cols,rows)` emitting the OSC control message (rate-limited); map close codes 4000‚Äì4003 to typed errors; reconnect with backoff + `kick()`; handle the `{recoverable, sessionState}` answer from `getPtyUrl` by surfacing a typed "needs resume" state.
*Accept:* unit tests for framing/close-code mapping; integration test: attach to a real task session, observe bytes, send `echo hi\n`, assert echo appears in stream.

**T1.4 ‚Äî `Dev3TerminalKit`: SwiftTerm integration** (L)
SwiftUI-wrapped SwiftTerm view: feed PTYClient output coalesced per `CADisplayLink` frame; input encoding incl. `Shift+Enter ‚Üí \x1b\r` (port `shift-key-sequences.ts` table where SwiftTerm differs); pinch-to-zoom ‚Üí font size ‚Üí refit ‚Üí resize message, double-tap reset, persisted preference; theme from Dev3UI (Tokyo Night / GitHub-light equivalents per `TerminalView.tsx` palettes); selection ‚Üí clipboard; `osc52Clipboard` push ‚Üí `UIPasteboard`. Compose-mode guard: terminal view never becomes first responder unless raw mode is on.
*Accept:* runs against a live task in the simulator (screenshot in PR); unit tests for key-encoding table and zoom‚Üíresize math; 5k-chunks/sec synthetic flood stays at 60fps (measure with a signposts test, document numbers).

**T1.5 ‚Äî Pairing UX** (M, depends T1.1)
QR scan (VisionKit `DataScannerViewController`, AVFoundation fallback), parse `<origin>/?token=<jwt>`, exchange, name-the-server flow; manual URL+code entry (dev/static-code path); Bonjour browse (after T0.4) listing discovered instances with "reconnect" using stored session; server list with active selection + delete (Keychain cleanup).
*Accept:* end-to-end pairing against `dev3 remote` on LAN in simulator (camera path manually QA'd on device later ‚Äî simulator uses manual entry); unit tests for URL/token parsing and store.

Integration checkpoints (single agent, after each wave): wire the wave's outputs together, run the live-server integration suite, update PROTOCOL.md if any discrepancy surfaced, and file follow-up tickets rather than patching in place.

## Standing risks the fleet must respect

1. **Protocol drift** ‚Äî backend PRs merge daily. Contract tests (T0.2) are the tripwire; any iOS PR that works around a server behavior instead of updating PROTOCOL.md + tests is wrong.
2. **Shared PTY min-size** (decision 060) ‚Äî do not "fix" client-side; v1 accepts + surfaces it, B3 fixes it server-side.
3. **Codex paste pacing** (decision 132) ‚Äî port `terminal-submit.ts` exactly; timing regressions are invisible in tests against plain shells.
4. **`~/.dev3.0/` layout invariants** ‚Äî backend tickets add files only; never rename/move (AGENTS.md hard rules).
5. **Apple-side unknowns** ‚Äî camera QR scanning and Local-Network permission behave differently on device vs simulator; T1.5/T4.1 include physical-device QA notes.
