Added two agent-CLI conveniences driven by platform vents. `dev3 note show <id>` prints a single note's complete body (8-char prefix, ambiguity/not-found errors) since `dev3 note list` only shows a truncated one-line preview — notes are the durable continuity record across context resets, so they must be readable in full. `dev3 current --brief` skips re-printing the full task description (which most agents already receive as their initial prompt), keeping project/task/status/overview to save context tokens. The dev3 skill teaches both.
