Generated Claude Code lifecycle hooks now treat the dev3 CLI's "app not running" exit code (2) as a non-blocking no-op, so editing files, running tools, submitting prompts, and stopping keep working when the desktop app is closed mid-session (e.g. during an app update). Claude Code reads hook exit code 2 as a blocking error, which previously wedged tool calls; the fix mirrors the selective fallback Codex hooks already use, so unrelated failures still surface.
