Added narrow-viewport carousel navigation for the Kanban board (Level 1). On a sub-768px viewport (phone via dev3 remote, or a narrow browser window — gated by a media query) the board shows one column at a time: swipe sideways or use the pager chevrons/dots to change column, scroll vertically to browse its tasks. Collapsed columns are excluded; empty columns are kept. Browser remote now renders at device-width so the layout is responsive. Hover terminal previews are disabled on touch and narrow viewports (no hover there, and the popover just covers the card). On narrow viewports the board's label-filter chip strip is replaced by a compact funnel button that opens a new bottom-sheet (so filtering stays reachable while reclaiming the vertical space), and opening a task zooms its workspace — the active-tasks sidebar/strip is hidden since there is no room for it. Introduced a reusable BottomSheet primitive (pure React, works in desktop and browser; backdrop/Esc/swipe-down dismiss, focus trap, safe-area, reduced-motion) plus a narrow-viewport clamp that keeps centered modals/palettes inside the phone viewport. On narrow viewports the global header's right-side action cluster (Quick Shell, Project Terminal, Remote access, Website/Report/Changelog, Project/Global Settings) folds into a single kebab that opens a BottomSheet, which also adds a touch entry to the Command Palette (otherwise keyboard-only, and the native menu is absent in remote); the prevent-sleep, git-pull, tmux and update-ready widgets stay inline. The same treatment now lands on the task screen: on narrow viewports the task info panel's two dense action toolbars collapse into a thin summary bar (status + title + diff badge) with a kebab that opens an actions BottomSheet holding every git/dev-server/ports/scripts/open-in/tmux action plus the full task details grid (rename, branch, PR, worktree, notes) — otherwise unreachable on a phone where the panel can't be resized. The same one-element-at-a-time idea now extends to the terminal itself (Level 2): on a narrow viewport a task opens with its tmux window zoomed to a single pane, and a compact top switcher — ‹ prev · a named-pane dropdown · next › — plus a horizontal swipe over the terminal, a horizontal trackpad scroll, or Arrow keys move between panes and re-zoom the target, so the agent/dev-server/shell panes are no longer crammed into a tiny split. The dropdown lists every pane by name (its explicit tmux title, else the running command) and renders above the terminal canvas — the glass switcher creates its own stacking context, so the carousel is isolated (`isolate` + z-layered) and the popover never falls behind the terminal. The swipe arbitrates by axis in the capture phase (a clearly-horizontal drag is the carousel; a vertical drag or tap falls through to the terminal untouched, so it never fights interactive TUIs, and any nascent text selection is collapsed at the gesture origin), zoom is enforced idempotently in one backend round trip via a new tmuxPaneNavigate RPC, and the switcher/gesture are absent for single-pane sessions. On the full (non-narrow) viewport a "Zoomed" badge appears at the top-right of the terminal whenever a multi-pane tmux window is zoomed (e.g. left over from a phone session, or ⌃B z) with a one-tap un-zoom — making the otherwise-invisible, shared per-window zoom state legible and reversible, without ever mutating it automatically. The carousel was also generalised into a product-wide narrow-viewport (mobile) UX doctrine in docs/ux/ (bible §12, ux-architecture responsive block, a UX decision, and feature-plans/narrow-viewport-doctrine.md) — one-element-at-a-time per surface, a reconciled 768px breakpoint, a swipe gesture law, a touch navigation/action-reachability model, and a mandated BottomSheet primitive. Full UX plan is documented in docs/ux/feature-plans/mobile-carousel-navigation.md. Idea by Ittai Zeidman.
