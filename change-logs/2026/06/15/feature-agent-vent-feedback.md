Added a background, local-only "vent" channel: agents run `dev3 vents "name" "markdown"` to file anonymous feedback about the dev3 platform itself — both bug reports (something in dev3 got in the way) and feature requests (a missing dev3 capability that would have saved work). Each vent is written as a standalone markdown file under ~/.dev3.0/vents/ with no project context, code, or PII attached — nothing leaves the machine and nothing changes in the UI. The dev3 skill always teaches agents the command and when to use it; the maintainer reads the folder directly.
