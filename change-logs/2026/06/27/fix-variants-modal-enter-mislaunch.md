Fixed the Launch Variants and Spawn Agent modals spawning agents when pressing Enter while the agent/config picker or the Watch/Cancel/Add/Remove buttons were keyboard-focused — Enter now only launches when no interactive control is focused. Added a reusable focus trap (useFocusTrap) to every modal/dialog (Launch Variants, Spawn Agent, About, Add Project, Create Task, Folder Picker, Task Detail, command/quick-switch palettes, Keyboard Shortcuts, Git Pull error, and the confirm dialog) so Tab/Shift+Tab cycle within the dialog instead of escaping to the Kanban board behind it, with focus restored on close. Added a global keyboard focus ring (`:focus-visible`) so Tab/Shift+Tab now visibly highlights the focused control (Select pickers, buttons, inputs) app-wide — shown only for keyboard focus, never on mouse click.
