Added a Google Analytics `agent_launched` event fired once per agent instance started, carrying the agent name and the chosen permission mode, plus an `agent_name` parameter on the existing `task_moved` event. Now usage can be broken down by which agents (Claude, Codex, Gemini, custom CLIs, etc.) people actually run and complete tasks with. Launch event is wired into every launch path: variants, retries, extra agent, and bug hunters. Analytics now also sends the user's public IP as `ip_override` (resolved best-effort once per launch and cached) so GA4 can populate the Country/City geo dimensions, which Measurement Protocol web hits otherwise leave unset.
