Added CLI commands that let agents drive the app UI: `dev3 notify` shows a clickable in-app toast (or a native OS notification with `--desktop`) that jumps you to the originating task and carries the source-task context (#seq · project · title); `dev3 attention "reason"` lights the red task-card badge and shows the reason(s) in the card's hover preview — repeated calls accumulate (newest 5 kept), with no native browser tooltip; and `dev3 ui state` reports the focused task/project, app foreground, how long the user has been idle (so an agent can pick the right channel or hold off), and the worktree's tmux layout (windows, panes, an ASCII pane map; `--json` for raw data). The dev3 agent skill documents when/how to use these so agents nudge on real blockers, questions, completions, and failures without spamming.
