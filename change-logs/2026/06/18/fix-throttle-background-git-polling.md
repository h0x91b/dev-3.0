Fixed UI freezes (main-loop stalls) caused by a storm of git processes. The background merge-detection and PR-promotion pollers now schedule each task individually: the project board you are actively viewing is polled at full cadence while every off-screen project is throttled heavily. Each check is given random jitter so equal intervals never align, fresh tasks are spread across their whole window, and a poll tick that arrives long after a laptop wakes from sleep re-spreads the backlog instead of firing every overdue check at once.
