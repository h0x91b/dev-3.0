The `bun run dev` script now binds the dev app's remote web UI to the task's first pool-allocated port (`DEV3_REMOTE_PORT=${DEV3_PORT0:-0}`), so the QA URL is derivable from `dev3 dev-server status` instead of scraping a random port or starting a separate `dev3 remote`. The dev-server now allocates pool ports on start (back-filling existing tasks), `portCount: 1` is committed for the project, and `dev3 config show` no longer hides unset config fields like `portCount`. A bare `bun run dev` still falls back to a random port.
