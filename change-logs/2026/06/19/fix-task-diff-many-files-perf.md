Sped up opening the task diff for large changesets (180+ files no longer take 10s+). The backend now reads every file's content through batched `git cat-file` calls and a single `git diff --numstat`, instead of spawning ~3-5 git processes per file; per-file hunks are dropped (the renderer derives them from content), turning hundreds of sequential git spawns into a handful. Reported and root-caused by Alexander Kiselyov, who dug into the sources and pinpointed the per-file git spawns.
