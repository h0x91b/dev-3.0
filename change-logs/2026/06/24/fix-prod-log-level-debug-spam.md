Backend file logging now defaults to `info` on prod/staging/canary builds (full `debug` is kept on dev builds and can be forced anywhere with `DEV3_LOG_LEVEL`). Previously the minimum level was hard-wired to `debug` and nothing ever lowered it, so every build wrote tens of thousands of trace lines per session to disk via synchronous appends — multi-MB/day logs and needless event-loop pressure on the busy git-status poll path. The highest-frequency poll traces (getBranchStatus / getResolvedProject / fetchOrigin / "No tasks file yet") were also demoted from `info` to `debug`.
