Added a virtual "Operations" board: a non-git, cross-project Kanban for ad-hoc, code-driven tasks (back up prod, stop a server, mail/Slack triage, throwaway experiments) that persist in history. Modeled as Project.kind "virtual" (stored in a separate ~/.dev3.0/virtual-projects.json so older app versions stay compatible), each operation runs an AI agent plus a split-right shell in a managed temp folder — or a folder you pick — with no git worktree, branch, diff, PR, or review columns. Create one from "Add Project → Operations"; completed operations stay in Done as history and are removed only when you delete them (a chosen folder is never auto-removed). The single home terminal is replaced by a built-in Operations board, and ⇧⌘` now opens a "Quick Shell" operation in your home directory. The built-in board is pinned first everywhere, reachable with ⌘0, and marked as a special system object (shown as "[ Operations ]" with a SYSTEM badge); inside an operation the inspector hides git, Dev Server, and Scripts, showing a subtle "Git is not available in operations tasks" note instead. Operations are now treated at full parity with git tasks across the app: a dead terminal session is restored (or relaunched) when you reopen an active operation, the dashboard shows operations' active tasks, the working-folder conflict warning works, and the dev3 CLI resolves and lists operations. A second sweep closed the remaining git-only gaps: the command palette no longer offers git, dev-server, or run-script actions for operations; the ⌘K project switcher pins the board first and badges it ⌘0 with its bracketed name; the branch-status poll (every 15s) and the open-PR poll (every 60s) are skipped for operations instead of spawning a doomed git; git RPCs (diff/rebase/merge/push/PR) reject operations with a clear message; cancelling an operation's preparation no longer attempts a git worktree removal; and offline `dev3 current` / short-id resolution read operations from virtual-projects.json. A third sweep fixed a real data-layer gap: creating or editing labels and custom columns on an Operations board (which route through updateProjectWith) no longer fails with "Project not found" — that helper now routes virtual ids to virtual-projects.json like the other mutators; the Operations board's Project Settings now show only the Board tab (labels + columns), hiding the git-only Project/Worktree config tabs; and the merge-notification cleanup also reaps virtual task ids so stale entries can't linger. A fourth sweep closed the last forgotten seams: opening the Project Terminal on an Operations board (header button, project tile, Cmd+`, command palette, or native menu) no longer crashes with "Project path does not exist" — the affordance and the Open-in-Finder tile button are hidden for virtual boards and the backend rejects the call; the pinned built-in Operations board can no longer be deleted (which previously dead-ended Cmd+0 until restart and orphaned its tasks); the dev3 CLI now resolves task context from the injected DEV3_TASK_ID for operations whose working dir is outside ~/.dev3.0/ops/ (fixed-folder ops and the Quick shell in $HOME), so agent status hooks stop silently no-opping there; the Quick shell hotkey reuses an inactive shell instead of spawning duplicates and is race-safe against rapid double-presses; and the Quick Shell header button and the scratch-session card badge now render their icon glyphs instead of an empty placeholder.
