Added a virtual "Operations" board: a non-git, cross-project Kanban for ad-hoc, code-driven tasks (back up prod, stop a server, mail/Slack triage, throwaway experiments) that persist in history. Modeled as Project.kind "virtual" (stored in a separate ~/.dev3.0/virtual-projects.json so older app versions stay compatible), each operation runs an AI agent plus a split-right shell in a managed temp folder — or a folder you pick — with no git worktree, branch, diff, PR, or review columns. Create one from "Add Project → Operations"; completed operations stay in Done as history and are removed only when you delete them (a chosen folder is never auto-removed). The single home terminal is replaced by a built-in Operations board, and ⇧⌘` now opens a "Quick Shell" operation in your home directory. The built-in board is pinned first everywhere, reachable with ⌘0, and marked as a special system object (shown as "[ Operations ]" with a SYSTEM badge); inside an operation the inspector hides git, Dev Server, and Scripts, showing a subtle "Git is not available in operations tasks" note instead.
