Fixed a false-positive "Branch Merged" prompt on PR-review tasks, on two layers. The merge-detection poller no longer compares a review branch against itself (it now falls back to the project's base branch, so a genuinely merged PR is still detected), and the GitHub PR fallback now requires the merged PR's head commit to equal the current local HEAD, so a stale/reused-name PR with new unmerged work no longer triggers the prompt.
