Added narrow-viewport carousel navigation for the Kanban board (Level 1). On a sub-768px viewport (phone via dev3 remote, or a narrow browser window — gated by a media query) the board shows one column at a time: swipe sideways or use the pager chevrons/dots to change column, scroll vertically to browse its tasks. Collapsed columns are excluded; empty columns are kept. Browser remote now renders at device-width so the layout is responsive. Hover terminal previews are disabled on touch and narrow viewports (no hover there, and the popover just covers the card). On narrow viewports the board's label-filter chip strip is replaced by a compact funnel button that opens a new bottom-sheet (so filtering stays reachable while reclaiming the vertical space), and opening a task zooms its workspace — the active-tasks sidebar/strip is hidden since there is no room for it. Introduced a reusable BottomSheet primitive (pure React, works in desktop and browser; backdrop/Esc/swipe-down dismiss, focus trap, safe-area, reduced-motion) plus a narrow-viewport clamp that keeps centered modals/palettes inside the phone viewport. On narrow viewports the global header's right-side action cluster (Quick Shell, Project Terminal, Remote access, Website/Report/Changelog, Project/Global Settings) folds into a single kebab that opens a BottomSheet, which also adds a touch entry to the Command Palette (otherwise keyboard-only, and the native menu is absent in remote); the prevent-sleep, git-pull, tmux and update-ready widgets stay inline. The carousel was also generalised into a product-wide narrow-viewport (mobile) UX doctrine in docs/ux/ (bible §12, ux-architecture responsive block, a UX decision, and feature-plans/narrow-viewport-doctrine.md) — one-element-at-a-time per surface, a reconciled 768px breakpoint, a swipe gesture law, a touch navigation/action-reachability model, and a mandated BottomSheet primitive. Full UX plan and the terminal pane carousel (Level 2, next step) are documented in docs/ux/feature-plans/mobile-carousel-navigation.md. Idea by Ittai Zeidman.
