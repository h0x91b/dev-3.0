Fixed `dev3 remote` dying on startup ("Unknown command: /$bunfs/root/dev3") on the compiled binary. Background mode (the default) re-spawns the server, and the child argv wrongly re-included bun's injected `/$bunfs/root/dev3` virtual entry path, which bun re-injects on the child too — shoving it into argv where it parsed as the command. The respawn argv computation now passes only the user-facing args (with the dev `bun run` script-path case handled), and a unit test covers the compiled-binary argv shape.
