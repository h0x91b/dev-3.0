Improved `dev3 remote` for remote-Linux use. Lifecycle: `--detach` runs the headless server in the background (surviving the SSH session), and new `dev3 remote status`, `dev3 remote url`, and `dev3 remote stop` subcommands let you query a running server, re-print a fresh QR/URL from any new SSH session, and shut it down cleanly. The server records its PID/port/socket under `~/.dev3.0/remote/` so a separate process can find and control it. On Linux, `dev3 remote install-service` installs and enables a systemd --user unit so the server survives logout and starts on boot (`uninstall-service` removes it). The browser now also remembers your session (localStorage, 8h) and silently reconnects on reload without rescanning the QR. In the Remote Access modal, when the tunnel is off, a new interface dropdown lets you choose which local IP (including localhost / 127.0.0.1) the QR and URL point at — handy for multi-homed boxes, VPNs, or the SSH-forward path. Hardened the lifecycle against a batch of edge cases: `install-service` now validates `--static-code` (length + no whitespace) at install time instead of silently restart-looping under systemd; `--detach` surfaces spawn failures cleanly and uses a lock file so two simultaneous launches can no longer orphan a server; the background server records its lifecycle state before (not after) the Cloudflare tunnel connects, so a blocked-outbound box no longer leaves the server invisible to `status`/`url`/`stop`; and the remote WebSocket proxy no longer leaks upstream connections that disconnect mid-handshake.
