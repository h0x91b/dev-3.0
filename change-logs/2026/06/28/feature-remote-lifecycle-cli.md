Added a lifecycle to `dev3 remote` for remote-Linux use: `--detach` runs the headless server in the background (surviving the SSH session), and new `dev3 remote status`, `dev3 remote url`, and `dev3 remote stop` subcommands let you query a running server, re-print a fresh QR/URL from any new SSH session, and shut it down cleanly. The server records its PID/port/socket under `~/.dev3.0/remote/` so a separate process can find and control it.
