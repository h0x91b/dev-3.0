Fixed a stale-lock break race in withFileLock where two processes sharing ~/.dev3.0/ could both break and acquire the same lock, risking silent data loss in tasks.json/projects.json. Stale locks are now claimed via an atomic rename + age re-validation (ABA-safe), and unexpected stat errors (e.g. EACCES) surface immediately instead of spinning forever. The on-disk lock format is unchanged, so older app versions stay interoperable.
