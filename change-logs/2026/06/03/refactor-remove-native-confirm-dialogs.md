Replaced native dialogs for remote/browser mode. (1) Confirm: swapped the native `Utils.showMessageBox` `showConfirm` RPC (with its `window.confirm` browser fallback) for an in-app promise-based React `confirm()` service; migrated all 10 call sites and removed the RPC. (2) Toast: replaced ~40 blocking `window.alert()` error popups and the old `ErrorToast` with a new imperative `toast.error/success/info/warning` service rendered by a single `ToastHost` (larger styling, variant colors, 30s auto-dismiss with a progress bar). (3) Menu dialogs: replaced the native `Utils.showMessageBox` boxes triggered from the macOS menu bar - "About" now opens an in-app React `AboutModal`, and the manual "Check for Updates" flow downloads silently (reusing the header "Update ready" plaque) and reports "up to date"/errors via toast, with no native confirm boxes. `Utils.showMessageBox` is no longer called anywhere. Documented a project-wide ban on native dialogs in AGENTS.md.
