Added multi-window support. File → New Window (or Cmd+Shift+N) opens another dev-3.0 window so users on multi-monitor setups can park a different project on each screen. Closing windows no longer quits the app — like a standard macOS app it keeps running in the dock even with no windows open, and clicking the dock icon brings a window back. Push messages, menu actions and the WKWebView resize nudge now route through a new window-manager module that broadcasts to all windows and targets the focused one for menu actions.

Unified the quit confirmation: a single `before-quit` gate now drives the existing React "sessions keep running" dialog for every quit path — Cmd+Q, the menu Quit item, and dock Quit — instead of only the renderer's Cmd+Q shortcut. The confirmation is React (works the same in the remote/browser client); the "don't show again" choice is persisted in global settings. Updater restarts bypass the gate so auto-update keeps working.

Fixed multi-window regressions: Cmd+Q now reliably starts the quit (the renderer catches it and forwards to the main process, since WKWebView swallows the native menu accelerator when a terminal is focused). Quitting while the app is window-less in the dock reopens a window and shows the confirmation dialog there (the reopened window pulls the pending state on mount, so the dialog reliably appears). Also fixed terminal sizing when the same task is open in two windows at once — the shared PTY now sizes to the smallest viewer (smallest-wins, larger windows letterbox) instead of flip-flopping geometry on every resize.
