Added multi-window support. File → New Window opens another dev-3.0 window so users on multi-monitor setups can park a different project on each screen. Closing a window no longer quits the app — the app exits only when the last window closes, matching standard macOS behavior. Push messages, menu actions and the WKWebView resize nudge now route through a new window-manager module that broadcasts to all windows and targets the focused one for menu actions.

Unified the quit confirmation: a single `before-quit` gate now drives the existing React "sessions keep running" dialog for every quit path — Cmd+Q, the menu Quit item, and closing the last window — instead of only the renderer's Cmd+Q shortcut. The confirmation is React (works the same in the remote/browser client); the "don't show again" choice is persisted in global settings. Updater restarts bypass the gate so auto-update keeps working.
