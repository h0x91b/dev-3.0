Fix dev-server start failing with "No dev script configured" when a new project keeps its `.dev3/config.json` only in a feature-branch worktree (not yet merged to the main branch). `resolveOperationalProjectConfig()` now falls back to the worktree-resolved script when the project-level script is empty or whitespace, instead of forcibly shadowing the worktree value with an empty string. Same fallback applied to `setupScript`, `devScript`, `cleanupScript`, and `setupScriptLaunchMode`.
