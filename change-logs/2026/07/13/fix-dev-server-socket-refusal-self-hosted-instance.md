Fixed the chronic dev-server stop/restart failures ("Empty response from server" / refused control socket, #910/#920): when a project's devScript boots a dev3 app of its own (dev-3.0 dogfooding), that guest instance won CLI socket discovery and killed itself mid-teardown before replying. The app now marks guest sockets with a meta sidecar and answers stop before tearing down its own host session, the CLI deprioritizes guest sockets and transparently replays idempotent dev-server commands against a surviving instance, so stop/restart are reliable and idempotent even with multiple app instances running.
