Fixed all new tmux panes (setup, rebase/git-op panes, splits, dev-server) starting in a wrong deleted directory after the Homebrew tmux 3.7 upgrade. The tmux server daemonized with the first task's worktree as its cwd; once that worktree was deleted, tmux 3.7 silently ignored `-c` everywhere. tmux clients now always spawn from the immortal `~/.dev3.0` and the pane cwd is passed explicitly via `-c`. Also fixed UI splits/new-window opening inside the .app bundle when tmux reports an empty `pane_current_path` — the `-c` format now falls back to `#{session_path}` (the task worktree).
