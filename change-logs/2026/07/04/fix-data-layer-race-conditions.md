Fix three task-lifecycle race conditions found in a bug-hunt pass. Guard-blocked `completed`/`cancelled` moves no longer tear down the worktree/PTY before the status guard check. The terminal-bell auto-transition now guards on still-in-progress inside the lock, so a late bell can't drag an already-reviewed task back to user-questions. `addAttempts` allocates variant indices atomically inside the task file lock, so concurrent calls can't mint duplicate variant numbers in one group.
