Mouse-copying terminal text no longer leaves the pane stuck in tmux scroll/copy mode. Reverted the copy-mode retention behavior so a mouse copy cancels copy-mode immediately and the terminal stays live — no more stray cursor jumps or having to press Escape after every copy. Copied text still reaches the clipboard. As a trade-off, copying older scrollback again snaps the viewport to live output; the underlying issue #978 is reopened to be solved a different way.
