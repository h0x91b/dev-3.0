Addressed a batch of iOS companion-app TestFlight feedback. Compacted the terminal composer by dropping the redundant "Insert / Compose mode" row (the mode is already shown by the accessory-bar toggle) and moving Insert next to Send under a full-width message field, reclaiming vertical space. Moved the Task Info Actions section (Complete/Cancel/Delete) directly beneath Overview so it is reachable near the top without scrolling. Fixed the diff view rendering every line of a modified file as a deletion — modified files without server hunks now compute a real line-by-line diff on-device (common prefix/suffix trim plus an LCS), so unchanged lines stay context. Added a stale-while-revalidate diff cache shared across navigation so switching diff modes or reopening a task's changes shows the last result instantly and refreshes in the background. Kept the terminal composer short in landscape (compact height) so the terminal keeps usable real estate instead of collapsing behind the input. Fixed tmux pane/window switches showing stale or blank output: the switch now forces a full server-side repaint of the shared session (only when the pane, zoom, or window actually changes), the iOS client no longer wipes its own screen on switch, and — the real root cause — the terminal frame buffer no longer kills its PTY-output consumer when a pane-switch discards a pending frame (it now keeps the producer alive instead of signalling cancellation, which had left the pane permanently blank). Fixed pinch-zoom in the terminal jumping the scroll position toward the bottom: the app now pins the tmux copy-mode top visible line just before the zoom resize so your place is preserved across the reflow. Reworded the notifications footnote to explain that notifications come directly from your Mac over a private live connection (no cloud), which is why they need that connection.
