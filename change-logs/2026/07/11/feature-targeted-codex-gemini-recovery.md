Session recovery now resumes the exact previous session for more agents. Gemini gets targeted recovery via its new launch-time --session-id flag (gemini-cli #26060), joining Claude and Cursor. Codex sessions are now recovered per-pane: dev3 records each pane's real Codex session id from the lifecycle hook (which reports the session id plus $TMUX_PANE), so reviving a worktree that ran several Codex agents (e.g. multiple bug hunters) resumes each pane to its own session instead of collapsing them onto the most-recent one. Note: Gemini --session-id requires a gemini-cli build from 2026-04-27 or later.
