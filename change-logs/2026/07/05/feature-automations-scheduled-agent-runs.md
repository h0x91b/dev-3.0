Added Automations — scheduled agent runs. Each project can define recurring schedules (RFC 5545 RRULE subset + IANA timezone) that fire a stored prompt with a chosen agent: every fire creates an ordinary task (worktree + tmux + agent) on the board, with a clock provenance glyph on the card. Runs are tracked (created / failed / missed); occurrences missed while the app was offline surface as a toast and can optionally catch up with a single run. Managed from a new Automations tab in Project Settings and via `dev3 automations` (list/show/create/update/delete/run/templates). Ships a built-in "What I shipped" weekly report template — a recurring AI digest of completed and in-flight work across projects.
