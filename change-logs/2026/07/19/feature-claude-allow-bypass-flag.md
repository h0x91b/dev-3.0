Claude sessions now always launch with --allow-dangerously-skip-permissions, so the dangerous bypass mode is available to toggle into (Shift+Tab) from any preset without being enabled by default. The flag is injected once by the claude adapter unless a bypass flag is already present, so it is never duplicated on top of the hard --dangerously-skip-permissions carried by Bypass/Accept-Edits presets. The Default presets no longer carry the hard bypass flag, so "Default" now opens in Claude's normal permission mode (bypass available but off) instead of full bypass; their versions were bumped so already-onboarded users get the corrected behaviour.
