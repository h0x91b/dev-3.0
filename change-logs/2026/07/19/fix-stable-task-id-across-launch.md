Task ids are now stable across launches: launching a To Do task (Launch Variants, scheduled launch) transforms it in place into variant #1 instead of deleting it and re-creating every variant under a fresh id, so the id printed by `dev3 task create` never dangles. The CLI additionally accepts a stable `--task seq:<N>` reference everywhere a task id is expected, and "Task not found" errors now explain that older launched ids may have been re-keyed and point at `dev3 tasks list`.
