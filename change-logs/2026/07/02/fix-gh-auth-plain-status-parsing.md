Fixed the "GitHub CLI (gh) is not signed in" false negative for gh versions (~2.50–2.63) that lack `gh auth status --json` but print the modern plain-text wording "Logged in to <host> account <login>" — the fallback parser only matched the old "as <login>" wording. The parser now accepts both wordings and marks the account flagged "Active account: true" as active instead of assuming the first listed one.
