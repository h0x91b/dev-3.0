Added `dev3 show-image <path> [--caption "..."] [<path> ...]` — agents can now surface images (screenshots, agent-browser captures, rendered charts) to the human in an in-app viewer bound to the task, with a clickable history (newest shown first). The viewer is a centred, windowed card (~3/4 of the screen) that clearly belongs to the task, with a fullscreen toggle (press `f`); images scale up to fill the frame and tall full-page captures switch to fill-width + vertical scroll. Each `--caption` annotates the image path it follows, so an agent can say what to look at in each shot — shown as a readable note below the image. New images raise the task's attention badge and a clickable toast; the viewer auto-opens only when you're already looking at that task (toggle in Settings → Behavior). Images are copied into the worktree `shared-images/` dir. Reopen the viewer anytime from the "Images" button (with a live count) in the task inspector's Runtime bar, next to Dev Server and Ports — it appears only when the task has shared images.
