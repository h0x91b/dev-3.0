Dev-server stop/restart now verifies teardown instead of fire-and-forget: processes are polled until actually dead (SIGTERM → SIGKILL escalation), daemonized devScript orphans holding the task's pool ports are found via port ownership and reaped, and stop waits for the ports to be released before returning. Start and status now surface "port already in use by PID/name" conflicts, and `dev3 dev-server start/restart --wait` blocks until the new server is really listening — so agents no longer test a stale bundle or hit invisible port collisions.
