Proved that the isolated native-session host can maintain a real Ghostty terminal screen while Bun.Terminal streams output: the terminal data callback now only enqueues raw bytes into a byte-capped queue and parsing runs on a deferred event-loop task, avoiding the Bun 1.3.14 Windows negative WASM allocation-pointer failure (preserved as a runnable regression probe). Parser-generated DSR/DA replies are written back to the same PTY so interactive TUIs keep working, and a fresh client reconstructs the same semantic screen after detach from a bounded, versioned parser-state snapshot with explicit overflow and failure verdicts. Opt-in and additive: the default host, protocol v1, production terminals, tmux, and persisted project/task data are unchanged.
