Fixed dev3's backend GitHub integration on machines authenticated only via a `GH_TOKEN` environment variable (e.g. Coder workspaces with GitHub external-auth, common in `dev3 remote`). dev3 now reads such an account's token straight from the environment instead of `gh auth token --user`, which has no stored credential to return and failed with "no oauth token found for … account …" — so pasting a PR URL into New Task, PR/merge detection, PR comments, and the open-PR button all work again. Stored-account token resolution additionally neutralizes any ambient token env var so it can't override the requested account.
