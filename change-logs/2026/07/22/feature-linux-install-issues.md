Short: dev3 remote on Linux

Polished the Linux + `dev3 remote` experience so dev3 runs cleanly on a fresh Linux box, fixing several issues hit while installing via Homebrew. The brew formula now ships the artifact-template next to the binary, so launching a task no longer fails with "Bundled dev3 artifact template not found" (a missing template also degrades gracefully now instead of blocking task launch). Both `dev3 remote` and the header Remote Access button now wait until the Cloudflare tunnel is actually reachable end-to-end — not just announced — before showing the public URL + QR, so the link resolves the instant you open it instead of dead-ending on a cached DNS_PROBE_FINISHED_NXDOMAIN. The tunnel is now marked connected only once cloudflared's own /ready endpoint reports a live edge connection (the CLI also shows progress dots and the modal keeps its "Starting tunnel…" state until then). The AI install guide (ai-install.txt) documents `brew trust` as a required step on Homebrew 6.0+ and hands the `dev3 remote` run-step off to the user rather than having the installing agent start the server itself. Adding a Codex account from a headless/remote box now generates a `codex login --device-auth` command (browser-less device flow) instead of the browser-only `codex login`, which is a dead end without a local browser.
