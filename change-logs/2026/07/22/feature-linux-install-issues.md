Short: dev3 remote on Linux

Polished the Linux + `dev3 remote` experience so dev3 runs cleanly on a fresh Linux box, fixing several issues hit while installing via Homebrew. The brew formula now ships the artifact-template next to the binary, so launching a task no longer fails with "Bundled dev3 artifact template not found" (a missing template also degrades gracefully now instead of blocking task launch). `dev3 remote` now waits until the Cloudflare tunnel is actually reachable end-to-end — not just announced — before printing the public URL + QR, with progress dots while it verifies, so the link resolves the instant you click it instead of dead-ending on a cached DNS_PROBE_FINISHED_NXDOMAIN. The AI install guide (ai-install.txt) documents `brew trust` as a required step on Homebrew 6.0+ and hands the `dev3 remote` run-step off to the user rather than having the installing agent start the server itself. Adding a Codex account from a headless/remote box now generates a `codex login --device-auth` command (browser-less device flow) instead of the browser-only `codex login`, which is a dead end without a local browser.
