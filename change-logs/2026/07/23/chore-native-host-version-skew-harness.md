Added an isolated proof harness for the tmux-removal roadmap that stages two immutable native terminal host images and proves the safe update rule: an existing session stays owned by its original compatible host, an incompatible client gets one explicit version-mismatch without killing or taking over the live session, a new session runs on the new image while the old one continues on the old image, and rollback selects a compatible staged image explicitly instead of guessing or falling back to tmux. Dev-internal only; no production code changed.
