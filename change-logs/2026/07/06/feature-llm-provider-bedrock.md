Added a per-agent LLM Provider toggle in Settings → Coding Agents — each agent can run on its native API (default) or a registered third-party backend (Amazon Bedrock for Claude). The toggle lives inside the agent's row and only appears for agents that have a backend. Selecting a third-party provider injects its environment (CLAUDE_CODE_USE_BEDROCK, mapped model id, geo) into that agent's launches and omits dev3's native --model alias, which the provider rejects with a 400. Known models map to provider-native ids automatically; a per-model override field handles region-pinned inference profiles, ARNs, or new models. Providers are defined in a registry, so new backends can be added with a single entry.
