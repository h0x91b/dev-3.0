Added an agent account switcher: Settings → Agent Accounts keeps multiple Claude Code and Codex logins and hot-swaps the active one without re-login. Claude accounts are isolated CLAUDE_CONFIG_DIR profiles (credentials per account, skills/settings/transcripts shared via symlinks) injected into new sessions; Codex accounts are auth.json snapshots swapped atomically with refreshed-token sync-back. Shows email/org/plan per account and flags out-of-band logins as unmanaged. Claude also supports API profiles — accounts backed by an API key / custom Anthropic-compatible endpoint (Anthropic API, OpenRouter, LiteLLM proxy, Bedrock via extra env vars) instead of a subscription login. The launch picker (Provider → Model → Mode) now shows a "via <account>" line under the Provider field whenever the selected provider has managed accounts, with a click-to-open popover for quick global switching without leaving the launch flow. Every switch asks for confirmation with an explicit billing warning (new sessions bill to the target account; running sessions are unaffected). An API profile's model now truly overrides the preset's --model flag, and the same login can be added from several organizations (duplicates are detected per account + organization). API profiles are now fully editable — the pencil on an API row reopens the whole form prefilled (base URL, key, models, extra env, label); the key is shown masked with a reveal toggle. Beyond a single "override all models" field, each Claude alias slot (Opus/Sonnet/Haiku/Fable) can be pointed at its own provider model id with an optional /model display name and description — dev3 rewrites the launch --model per family and exports the ANTHROPIC_DEFAULT_*_MODEL vars for aliases, background tasks and subagents. Each slot's display name auto-fills from the model id's last path segment (editable), and every field shows a realistic example placeholder. Every field carries an (i) tooltip, and the switch warning now reminds you that the account only applies to agents dev3 launches (Spawn agent, Find bugs, …), not a hand-run `claude`. Switching from an API profile back to a normal login now actively unsets the profile's ANTHROPIC_BASE_URL / ANTHROPIC_API_KEY / model vars in new sessions (they previously leaked through the long-lived tmux server env and hijacked OAuth sessions).
