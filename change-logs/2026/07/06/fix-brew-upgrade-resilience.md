Survive Homebrew fighting the in-app updater: the app now chdirs to ~/.dev3.0 at startup so a `brew upgrade` that replaces (or rips away) the .app bundle can no longer break child process spawning (tmux ENOENT) in the running instance, and the Homebrew cask is marked `auto_updates true` so a bulk `brew upgrade` skips dev3 instead of colliding with in-app-updated installs (explicit `brew upgrade --cask dev3` still works). Also adds `dev3 doctor` — a read-only install health check (app bundle, tmux shim, pinned tmux keg, Homebrew cask/formula state) that works without the app running and prints the exact fix command for anything broken — plus a README troubleshooting section covering the same recovery scenarios.
