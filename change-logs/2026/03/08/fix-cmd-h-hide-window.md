Fix Cmd+H not hiding the app on macOS. WKWebView (used by Electrobun) intercepts keyboard events before they reach the native menu system, so the `{ role: "hide" }` menu item's Cmd+H accelerator was silently swallowed. Added a JavaScript-level capture-phase handler that intercepts Cmd+H and calls the new `hideApp` RPC handler, which uses osascript to invoke macOS System Events hide. Closes #162.
