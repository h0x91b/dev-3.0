When clicking the Dev Server button while a dev server is already running, the app now shows a dropdown menu with Restart and Stop options. The dev server runs in a dedicated tmux session (`dev3-dev-<taskId>`) with a viewer pane nested inside the task session via `TMUX= tmux attach-session`. Server detection uses `tmux has-session` so it works correctly after app restarts. Dev server sessions are hidden from the tmux session manager UI and are cleaned up on task completion, cancellation, and manual stop. Fixed a restart race condition where a `trap EXIT` in the viewer pane shell could fire after the new session was already created (same name); the fix tracks the viewer pane ID explicitly and kills it before recreating the dev session.
