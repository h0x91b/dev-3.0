Fixed orphaned tmux sessions showing a raw `dev3-XXXXXXXX` name with no title or navigation in the tmux session manager. Root cause: `deleteTask` only killed the tmux session for active tasks, leaving zombie sessions when completed/cancelled tasks were deleted. Now `deleteTask` always attempts a best-effort session kill regardless of status. The UI also shows an "orphaned" badge and strips the `dev3-` prefix for sessions whose task data no longer exists.
