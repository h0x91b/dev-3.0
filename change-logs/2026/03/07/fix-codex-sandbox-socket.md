Fix dev3 CLI connectivity when running inside Codex seatbelt sandbox. The sandbox blocks `process.kill(pid, 0)` signals to external processes (EPERM) and Unix socket `connect()` calls (ENOENT). Added `ensureCodexConfigFile()` that auto-patches `~/.codex/config.toml` on app startup with `[permissions.network]` settings to allowlist the dev3 socket path. Also fixed `discoverSocket()` to treat EPERM as a sandbox indicator instead of a dead process, and fixed a pre-existing bug where `task show` displayed description even when identical to title.
