Fixed a bug where `dev3 task create --title "..." --description @file` would silently discard the description. The CLI socket handler for `task.create` was passing the title as the description to `data.addTask()` and ignoring the actual description parameter. Now the full description is stored as the task body, and the title is saved as `customTitle`. Also fixed `--limit` in `tasks list` being sent to the server but never applied — it now works client-side. Fixed pre-existing test failures where "In Progress" status label was renamed to "Agent is Working" but tests weren't updated. Added missing handler tests for `task.update` (clearing customTitle, description-only update with existing customTitle) and `task.move` (cancelled → active reopen).
