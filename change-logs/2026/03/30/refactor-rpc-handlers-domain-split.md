Refactored Bun RPC handlers out of the monolithic `src/bun/rpc-handlers.ts` file into domain modules for app/project, settings/config, task lifecycle, git operations, tmux/PTY, notes/labels, and remote access. Kept the public `rpc-handlers.ts` API as a thin composition layer so existing imports and tests keep working while handler ownership is obvious. Fixed regressions in `getPtyUrl` session restore and in background variant/attempt preparation so failed config resolution clears `preparing` state instead of leaving tasks stuck and throwing unhandled rejections.
