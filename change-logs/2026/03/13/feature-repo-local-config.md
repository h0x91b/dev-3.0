Project settings (setup/dev/cleanup scripts, clone paths, base branch, peer review toggle) now live exclusively in `.dev3/config.json` (shared, committed to git) and `.dev3/config.local.json` (personal, git-ignored). Settings in `projects.json` are no longer used — that file only stores project metadata. Existing settings are auto-migrated to `.dev3/config.json` on first load. Project Settings UI has two tabs: "Repo Config" and "Local Overrides". A `dev3-project-config` agent skill teaches AI agents about the config schema and when to create/modify these files.
