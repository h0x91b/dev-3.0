Project settings (setup/dev/cleanup scripts, clone paths, base branch, peer review toggle) now live exclusively in `.dev3/config.json` (shared, committed to git) and `.dev3/config.local.json` (personal, git-ignored). Settings in `projects.json` are no longer used — that file only stores project metadata. Existing settings are auto-migrated to `.dev3/config.json` on first load. Project Settings UI has two tabs: "Repo Config" and "Local Overrides". A `dev3-project-config` agent skill teaches AI agents about the config schema and when to create/modify these files. Config is now correctly resolved from the active worktree (not just project root), so `.dev3/config.json` created on a feature branch is visible to the Dev Server button and CLI. The disabled Dev Server button now shows a clickable hint popover with a copy-pasteable prompt to tell your AI agent to configure the dev script. Also added an "Install dev3 CLI" button in Global Settings for developers to symlink the CLI from the current app instance.
