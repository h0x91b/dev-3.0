Fixed Shift+Tab not working in the terminal. ghostty-web's InputHandler has a bug where both Tab and Shift+Tab produce the same `\t` character instead of sending the correct CSI Z (`\x1b[Z`) backtab sequence for Shift+Tab. Added a `attachCustomKeyEventHandler` workaround in TerminalView to intercept Shift+Tab and send the proper escape sequence. This fixes mode switching in tools like claude-code that rely on Shift+Tab.
