Add copy-on-write clone paths for worktrees. Users can now configure relative paths (e.g. `node_modules`, `.venv`) in Project Settings that get automatically cloned from the project root into new worktrees using CoW. On macOS, uses `clonefile()` syscall via Bun FFI for atomic whole-tree cloning (~10x faster than cp), with cascading fallbacks to `cp -cR` and `cp -R`. On Linux, tries `cp --reflink=always` (btrfs/xfs) before falling back to regular copy. All paths are cloned in parallel, failures are non-blocking, and invalid/missing paths are gracefully skipped.
