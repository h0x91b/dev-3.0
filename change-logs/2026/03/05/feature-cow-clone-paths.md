Add Copy-on-Write file/directory cloning for worktree setup. Projects can now configure clone paths (e.g. `node_modules`, `.venv`) that are cloned from the root project into each worktree using APFS `clonefile(2)` on macOS (with `cp -cR` and regular copy fallbacks) or `--reflink` on Linux. Clone paths are auto-detected from a comprehensive list of well-known directories (JS, Python, Ruby, Go, Rust, Java, C++, .NET, PHP, Elixir, etc.) when a project is added. Glob patterns (`*.cache`, `src/**/node_modules`) are supported. Project Settings UI includes an Auto-detect button with feedback, which also runs automatically when clone paths are empty.
