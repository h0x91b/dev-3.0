Fix Codex 0.131+ rejecting `--profile dev3-light` due to legacy `[profiles.dev3*]` blocks in `~/.codex/config.toml`. On new Codex versions, dev-3.0 now writes per-profile settings into `~/.codex/dev3.config.toml`, `dev3-light.config.toml`, and `dev3-dark.config.toml`, removes the legacy blocks and any managed top-level `profile = "dev3*"` selector from the main config, while older Codex versions keep the original in-main-config form.
