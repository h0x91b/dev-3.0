Added an icon-only Scripts button (ƒ function glyph) in the task header — placed between Kill tmux and Dev Server — that opens a dropdown listing all package.json scripts in the worktree. Click a script and a 5-button placement picker replaces the list — pick left, top, right, bottom or new tab and the script opens in a fresh tmux pane (or window) while the dropdown closes. The picker pre-selects the placement last used for that script (so a second launch is just Enter) and supports keyboard navigation (←/→ to choose, Enter to launch). The runner (bun/pnpm/yarn/npm) is auto-detected from lockfiles and a multi-lockfile warning is shown when ambiguous. Scripts are sorted by most recently launched first so the workflow-relevant ones float to the top. A fuzzy search input is focused on open so users can just type to narrow down the list; arrow keys move between matches and Enter opens the placement picker for the highlighted script. The header was also trimmed elsewhere: the "Files" button now shows only its folder icon to save horizontal space. Native menu adds Task → Run Script… (⌘R). Each launch is fire-and-forget — running the same script twice opens two panes, which the user can close in tmux when done.
