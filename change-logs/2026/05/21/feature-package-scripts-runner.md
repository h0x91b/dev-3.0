Added a Scripts button next to Dev Server in the task header that opens a dropdown listing all package.json scripts in the worktree. Click a script and a 5-button placement picker replaces the list — pick left, top, right, bottom or new tab and the script opens in a fresh tmux pane (or window) while the dropdown closes. The runner (bun/pnpm/yarn/npm) is auto-detected from lockfiles and a multi-lockfile warning is shown when ambiguous. Scripts are sorted by most recently launched first so the workflow-relevant ones float to the top. A "Run multiple scripts…" batch modal lets several scripts be launched at once, each into its own pane. Native menu adds Task → Run Script… (⌘R) and Run Multiple Scripts… (⇧⌘R). Each launch is fire-and-forget — running the same script twice opens two panes, which the user can close in tmux when done.
