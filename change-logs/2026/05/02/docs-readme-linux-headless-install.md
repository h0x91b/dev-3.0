Rewrote the README install section. Split macOS desktop app from Linux server (headless), and added a `dev3 remote` build-from-source recipe for Debian/Ubuntu — system deps, optional 4 GB swap on 2 GB VMs, Bun, and a single `./dist/dev3 remote` invocation that prints its own QR code and access URL. Forcing the build through `bun --bun ./node_modules/vite/bin/vite.js` avoids the Node OOM that hits when both runtimes are installed. Validated end-to-end on a fresh Hetzner CPX11 (Ubuntu 24.04) — UI loads in the browser at the printed URL.

Also fixed a remote-mode bug in `getProjectPtyUrl` browser override (`src/mainview/rpc.ts`): clicking "Project Terminal" produced "Failed to connect to PTY server" because the override was missing — the raw `ws://localhost:<ptyPort>` URL from the backend was sent to the laptop browser, where `localhost` resolves to the laptop, not the server. Added the override so the URL routes through `/pty?session=project-<id>&token=<jwt>` like every other PTY URL in browser mode.
