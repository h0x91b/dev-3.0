Fixed `DEV3_TASK_ID` (and other per-task env vars) leaking between sibling tasks. The tmux server inherits its global env from whichever client starts it first, so new sessions/panes saw the first task's id — causing docker containers built from `$DEV3_TASK_ID` to mount against the wrong worktree. Per-task env is now passed atomically via `-e KEY=VAL` to every dev3-issued tmux entrypoint (`new-session`, `split-window`, `new-session -d`), with the existing post-spawn `set-environment` kept as a fallback for the `-A` attach path.

Also tightened the agent skill: `split-window` is now the explicit default for background processes (celery, docker exec, log tail, watchers, dev servers). `new-window` is reserved for explicit user requests like "open a tab" — agents were creating hidden tabs for celery workers that users had to click to find.
