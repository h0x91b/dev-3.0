Fix reviving a task from completed/cancelled back to in-progress. After PR #571 the cleanup script was silently skipped on every task-to-done move (Bun.file(dir).exists() always returns false for directories), and removeWorktree would abort before deleting the branch if the worktree directory was missing — both leaving stale branches/worktree dirs behind so the next reactivation failed with "branch already exists" or "directory already exists". Restored existsSync for the directory check, made removeWorktree tolerate a missing worktree dir, and added self-healing in createWorktree so reactivation can reclaim stale leftovers automatically.
