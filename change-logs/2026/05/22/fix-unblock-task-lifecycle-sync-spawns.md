Unblocked the task lifecycle hot path: `pty.destroySession` now kills the tmux session via async spawn (was sync), `getUserShell` caches the resolved login shell across the app lifetime instead of running `dscl` on every task launch, and the `which tmux` diagnostic is logged once per app run rather than on every PTY creation. These sync spawns were freezing the Bun event loop while moving tasks to done/cancelled, blocking the UI and forcing `kill -9 dev3`.
