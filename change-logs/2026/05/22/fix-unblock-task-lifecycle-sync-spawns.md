Unblocked the task lifecycle hot path by removing every sync spawn from PTY create/destroy: `destroySession` kills the tmux session via async spawn, `configureTmux`/`setupTiledLayout`/`applyTmuxTheme`/`capturePane`/`tmuxSessionExists`/`splitAndRunCommand`/`listPaneIds` are now async, the `which tmux` diagnostic is fire-and-forget, the `existsSync(cwd)` pre-flight is dropped, and `runCleanupScript` uses `Bun.file().exists()`. `getUserShell` caches the resolved shell for 1h instead of running `dscl` on every task launch. Added a `loop-monitor` heartbeat that logs whenever the Bun event loop is blocked for >500ms, plus step-by-step debug logs around `Bun.spawn` and the first PTY output so future freezes are diagnosable from the log.
